♥ Business Model Pivot

0 / 0 ▼
From the research: Duolingo gates convenience, not content. All content should be free. You monetize by removing friction (hearts, ads, streak freeze). This is the #1 strategic change.
  • Build Hearts / Lives System Critical Large
    Free users get 5 hearts. Each wrong answer costs 1 heart. At 0 hearts: wait to recharge (1 heart / 4-5 hours) OR practice old lessons to earn hearts back. Pro users get unlimited hearts. This is Duolingo's #1 conversion driver.
    New: useHeartsStore or extend useEngagementStore • UI: HeartsDisplay component • Touch: QuestionCard, SessionView, access-control.ts
  • Remove Content Gating — Make ALL Units Free Critical Medium
    Currently Free tier locks units 2-10. Change so all units and all lessons are accessible to everyone. Remove "Unit 1 only" restriction from pricing.ts, access-control.ts, and all UpgradeGate/UpgradeModal references gating content.
    pricing.ts (FREE_TIER.maxUnits) • access-control.ts (canAccessUnit) • UpgradeGate.tsx • UpgradeModal.tsx • useCourseStore.ts
  • Remove Daily Question Limit Critical Small
    Replace the "5 questions/day" free-tier limit with the hearts system. The hearts mechanic is the new rate limiter. Remove daily_usage counting logic and DailyLimitBanner.
    pricing.ts • access-control.ts (canStartPracticeSession, getRemainingDailyQuestions) • DailyLimitBanner.tsx • daily_usage table queries
  • Remove Practice Mode Gating High Small
    Adaptive, interview-sim, and weak-areas modes are currently Pro-only. Make all practice modes free. The hearts system is the only friction lever now.
    pricing.ts (FEATURES) • access-control.ts (canAccessPracticeMode) • practice mode selectors
  • Update Pro Perks to Convenience-Based High Medium
    Pro should offer: unlimited hearts, no ads (future), 1 free streak freeze/day, 2x XP on weekends, premium league rewards, detailed progress insights. Update pricing.ts FEATURES, pricing page, and all gating logic.
    pricing.ts • pricing/page.tsx • UpgradeModal.tsx • shop items • engagement store
  • Update Pricing: $7.99/mo · $49.99/yr High Small
    Change from $9/mo & $79/yr to $7.99/mo & $49.99/yr. Update Paddle price IDs, pricing page copy, and all checkout references. Lower price point + convenience-gating = higher conversion.
    pricing.ts • pricing/page.tsx • Paddle dashboard • .env price ID vars
  • Update Pricing Page & Upgrade Modals High Medium
    Rewrite pricing page to show the new Free vs Pro comparison: all content free on both tiers, Pro = unlimited hearts + no ads + streak freeze + 2x XP + premium stats. Update all UpgradeModal/UpgradeGate messaging.
    pricing/page.tsx • UpgradeModal.tsx • UpgradeGate.tsx • checkout flow

  • Tighten Onboarding → First Lesson Pipeline Critical Medium
    After signup, user should immediately start their first lesson (not land on an empty dashboard). Pattern: Sign up → pick a starting unit → first lesson auto-starts → earn XP → see celebration → streak begins. Minimize clicks to first "win."

  • Pro Trial Prompt After First Lesson High Medium

    New: TrialPromptModal.tsx • LessonView completion handler • Paddle trial config
  • Landing Page → Auth Flow Polish Medium Small

🚀 Post-Launch & Future (Not Blocking)

0 / 0 ▼
These items are not required for launch but should be on the roadmap for the first few months post-launch.
  • Ads Integration (When 10K+ DAU) Low Large
    Interstitial ads between lessons for free users. Not worth building until scale. Serves dual purpose: revenue ($0.02-0.05/DAU/day) + conversion lever (annoying enough to drive upgrades). Only 7% of Duolingo revenue comes from ads.
  • Rename: MechReady → Multi-Topic Brand Low Large
    Per market research: when expanding to Personal Finance (Phase 1 expansion), rename to a topic-agnostic brand (Knowara, Thinkdex, etc.). ~195 occurrences across ~55 files. High-risk: localStorage key migration needed. Can launch as MechReady first.
  • Multi-Topic Architecture (Course Picker) Low Large
    Add a topic/profession picker like Duolingo's language picker. Each topic has its own course map, units, and questions. The gamification engine (XP, streaks, leagues) is shared. First expansion topic: Personal Finance.
  • Email Verification Flow Low Medium
    Currently no email verification after registration. Not blocking for launch (most users will use Google OAuth), but add verification for email/password accounts to reduce spam signups.
  • E2E Tests (Playwright) Low Large
    No E2E tests currently. Add Playwright tests for critical paths: signup, first lesson, purchase flow, streak tracking. Prevents regressions as the codebase grows.
  • Multi-Device Sync Low Large
    Currently client-first with server sync. Two devices can diverge. Need conflict resolution strategy: last-write-wins or merge. Important when mobile app ships, not critical for web-only launch.
