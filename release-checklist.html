♥ Business Model Pivot

0 / 0 ▼
Strategic decision required: Your market research says Duolingo gates convenience, not content. Currently your app gates content (unit 1 only) AND has hearts. The research recommends making all content free and letting hearts be the sole friction lever. These items implement that shift. Skip this section if you want to launch with the current gating model.
  • Make ALL Units Free Critical Medium
    Change LIMITS.free.unlockedUnits from [0] to 'all'. Update canAccessUnit() in access-control.ts. Remove or repurpose UpgradeGate/UpgradeModal references that block unit access. Hearts become the sole friction.
    pricing.ts • access-control.ts • UpgradeGate.tsx • UpgradeModal.tsx
  • Remove Daily Question Limit Critical Small
    Hearts already limit wrong answers. The 5-question/day cap is redundant and frustrating. Set LIMITS.free.dailyQuestions to -1. Remove or hide DailyLimitBanner. Keep daily_usage table for analytics but stop enforcing it.
    pricing.ts • access-control.ts • DailyLimitBanner.tsx
  • Make All Practice Modes Free High Small
    Remove adaptive, interview-sim, weak-areas from PRO_SESSION_TYPES. Update canAccessPracticeMode(). Let free users access everything — hearts are the only gate.
    pricing.ts (PRO_SESSION_TYPES) • access-control.ts
  • Update Pricing: $7.99/mo · $49.99/yr High Small
    Change TIERS.pro.priceMonthly from 900 to 799, priceYearly from 7900 to 4999. Create matching Paddle products in dashboard. Update env vars with new price IDs.
    pricing.ts • Paddle dashboard • .env price IDs
  • Rewrite Pricing Page for New Model High Medium
    Current page lists "5 questions/day" and "Unit 1 only" as free limitations. Rewrite to show: Free = all content + 5 hearts, Pro = unlimited hearts + streak freeze + 2x XP weekends + detailed stats. Emphasize the "all content is free" angle.
    pricing/page.tsx • UpgradeModal.tsx copy
  • Update Landing Page Copy High Medium
    If get-started page mentions content limitations ("Unit 1 free"), update messaging to: "Learn mechanical engineering for free. All lessons, all topics. Upgrade for unlimited hearts." Sell the free-everything angle.
    get-started/page.tsx • components/landing/

⚡ Onboarding & Conversion

0 / 0 ▼
80-90% of Duolingo's conversions happen on Day 0. Your onboarding currently just redirects to home. The path from signup to "first dopamine hit" needs to be shorter.
  • Onboarding → First Lesson Pipeline Critical Medium
    Currently /onboarding just redirects to /. After signup, user should pick a starting unit and immediately begin their first lesson. Minimize clicks between "I just registered" and "I earned my first XP."
    onboarding/page.tsx • (app)/page.tsx • useCourseStore
  • Pro Trial Prompt After First Lesson High Medium
    After completing their first lesson (dopamine peak), show a non-intrusive "Try Pro free for 7 days" modal. Highlight unlimited hearts. Configure Paddle trial period.
    New: TrialPromptModal.tsx • LessonView completion handler • Paddle trial config
  • Delayed Push Notification Prompt Medium Small
    Don't ask for push permissions on first visit. Wait until user has 2-3 lessons done and a streak to protect. "Want us to remind you to keep your streak?" gets much higher permission grant rate.
    PushPrompt.tsx • trigger logic in course/engagement store

☑ Manual Testing

0 / 0 ▼
  • npm test — All Tests Pass Critical Small
    Run npm test. Fix any failures in the ~25 test files across achievements, gems, leagues, lessons, streaks, and XP.
  • Full User Journey Smoke Test Critical Medium
    Walk through as a new user: Landing → Register → First lesson → Earn XP → Lose hearts → See OutOfHeartsModal → Hearts recharge → League placement → Quest completion → Visit shop → Invite friend. Every step smooth.
  • Payment Flow E2E (Paddle Sandbox) Critical Small
    Test: Checkout opens → payment succeeds → webhook fires → subscription in DB → hearts become unlimited → all Pro perks active. Then test cancellation flow.
  • Mobile Responsiveness Check High Medium
    Test on real phone or DevTools (375px, 414px): course map, lesson view with SVG diagrams, pricing page, quiz cards (especially fill-in-blank), settings, league board. Fix any overflow or broken layouts.
  • Cross-Browser: Chrome + Safari + Firefox Medium Medium

  • Content Correctness Spot-Check High Large
    Play through 3-5 lessons in different units. Verify: correct answers are correct, explanations make sense, SVG diagrams display properly, no duplicate questions. Wrong answers in an education app destroy trust.
